ช่วงนี้กระแสความกลัวว่า AI จะทำให้อาชีพนักพัฒนาซอฟต์แวร์หายไปเกือบหมด กลายเป็นเรื่องที่ถูกพูดถึงกันแทบทุกวัน บางคนถึงขั้นเชื่อว่าอีกไม่กี่ปีนักพัฒนาจะหายไป 80-90% เพราะ AI สามารถ generate code ได้เอง เขียน test ได้เอง หรือแม้แต่ deploy ระบบได้เอง<br/>

แต่ส่วนตัวผมคิดว่าภาพนั้น "เกินจริง" ไปมาก<br/>

สิ่งที่จะเกิดขึ้นจริงน่าจะเป็นการลดจำนวนคนลงในช่วงแรกมากกว่า โดยเฉพาะงานที่เป็น repetitive task หรือ workflow ที่สามารถ automate ได้ง่าย บริษัทจำนวนมากจะเริ่มใช้คนต่อ project น้อยลง จากเดิมที่ต้องใช้ทีม 10 คน อาจเหลือ 5-6 คน หรือ startup ที่เคยต้องจ้าง backend, frontend, devops แยกกัน ก็อาจใช้คนแค่ 1-2 คนที่ทำงานร่วมกับ AI ได้ดีพอแทน<br/>

แต่ในระยะยาว ผมกลับมองว่าปริมาณ “งานพัฒนาซอฟต์แวร์” จะเพิ่มขึ้นเร็วกว่าความสามารถในการลดคนเสียอีก<br/>

ที่ผ่านมา ซอฟต์แวร์ ถูกจำกัดอยู่แค่ในคอมพิวเตอร์ มือถือ หรือระบบเฉพาะทางบางอย่าง แต่ในอีก 10-20 ปีข้างหน้า ซอฟต์แวร์ จะเข้าไปอยู่ในทุกอย่างรอบตัวมนุษย์มากกว่าที่เราเคยเห็นมาก่อน<br/>

ทีวีจะไม่ใช่แค่ทีวีอีกต่อไป<br/>
ตู้เย็นจะไม่ใช่แค่เครื่องทำความเย็น<br/>
ซอฟต์แวร์ในรถยนต์จะไม่ใช่แค่ของประดับ<br/>
แม้แต่เตาอบ เครื่องซักผ้า หรือเครื่องใช้ไฟฟ้าธรรมดา ก็จะมีความซับซ้อนของซอฟต์แวร์ภายในมากขึ้นเรื่อยๆ<br/>

คนรุ่นถัดไปจะมอง "ทีวีที่ไม่มี software update" เป็นของแปลกพอๆกับที่คนยุคนี้มองโทรศัพท์ไม่มีอินเทอร์เน็ต<br/>

Software is eating the World กำลังเกิดขึ้นจริงด้วยความเร็วที่คาดเดาไม่ได้อีกแล้ว ไม่ใช่แค่ในความหมายของ startup หรือ web application ด้วย มันจะลามเข้าไปถึงทุกอย่าง appliance, infrastructure, healthcare, transportation, education และ workflow ของแทบทุกอุตสาหกรรม<br/>

เมื่อซอฟต์แวร์แทรกเข้าไปในทุกอย่าง ความต้องการคนที่เข้าใจการสร้าง ดูแล ปรับแต่ง และเชื่อมต่อระบบเหล่านี้ก็เกิดขึ้นมหาศาล<br/>

สิ่งที่เปลี่ยนคือ “รูปแบบของนักพัฒนา”<br/>

นักพัฒนารุ่นใหม่จะไม่ได้นั่งเขียน code ทีล่ะบรรทัดอีกแล้ว แต่จะกลายเป็นคนที่ทำงานร่วมกับ AI เพื่อออกแบบ workflow, validate logic, เชื่อมระบบ, ดูแล infra, วิเคราะห์ requirement และตัดสินใจเชิงเทคนิคแทน<br/>

พูดง่ายๆคือ AI จะลด "แรงงานเชิงกล" ของงานพัฒนาซอฟต์แวร์ แต่ไม่ได้ลดความต้องการคนที่เข้าใจซอฟต์แวร์จริงๆ<br/>

และถ้ามองเทียบกับสายอาชีพอื่น ผมกลับคิดว่านักพัฒนาซอฟต์แวร์อาจโดนกระทบน้อยกว่าหลายสายด้วยซ้ำ<br/>

อาชีพอย่าง นักวิเคราะห์, นักกฎหมาย, นักทำบัญชี หรือสายเอกสารจำนวนมาก มี workflow ที่เป็น structured reasoning สูงแต่ก็เป็นสิ่งที่ AI เก่งขึ้นเร็วมากในช่วงไม่กี่ปีที่ผ่านมา งานจำนวนมากในสายเหล่านี้จะถูกลด head-count หนักกว่านักพัฒนาด้วยซ้ำ โดยหลายคนในสายงานเหล่านั้น ก็อาจต้อง pivot เข้ามาใกล้งานพัฒนาซอฟต์แวร์มากขึ้น<br/>

เราอาจเริ่มเห็นนักทำบัญชีที่ต้องเขียน automation เอง<br/>
นักกฎหมายที่ต้องใช้ AI pipeline วิเคราะห์คดี<br/>
นักวิเคราะห์ที่ต้อง build internal tooling ของตัวเอง<br/>
หรือคนสาย domain expert ที่ต้องเรียนรู้การพัฒนาซอฟต์แวร์ในระดับหนึ่งเพื่อทำงานร่วมกับ AI ได้<br/>

พูดอีกแบบคือ โลกไม่ได้กำลังเดินไปสู่ "โลกที่ไม่มีนักพัฒนา"<br/>

แต่มันกำลังเดินไปสู่โลกที่ "คนแทบทุกสายต้องเข้าใจซอฟต์แวร์มากขึ้น"<br/>

ดังนั้นถ้าคุณอยู่ในสายงานพัฒนาซอฟต์แวร์อยู่แล้ว ผมคิดว่าไม่จำเป็นต้องมองโลกในแง่ร้าย<br/>

สิ่งที่ควรทำมากกว่าคือเริ่มยอมรับว่าโลกการทำงานกำลังเปลี่ยน<br/>
AI จะกลายเป็นส่วนหนึ่งของ workflow ปกติ<br/>
ความเร็วในการทำงานจะสูงขึ้น<br/>
ความคาดหวังจะสูงขึ้น<br/>
การแข่งขันจะสูงขึ้น<br/>

แต่ในขณะเดียวกัน ปริมาณซอฟต์แวร์บนโลกก็จะเพิ่มขึ้นแบบมหาศาลเช่นกัน
งานของนักพัฒนาจะง่ายขึ้นสำหรับความซับซ้อนของระบบที่มีอยู่ แต่ขอบเขตของความซับซ้อนนั้นกำลังถูกผลักออกไปไกลขึ้นเรื่อยๆ
เดาไม่ออกเหมือนกันว่าภายใน 10 ปีข้างหน้าจะเกิดอะไรขึ้นบ้าง ความซับซ้อนของงานจะมากขนาดไหนแล้วนักพัฒนาต้องเก่งแบบไหน แต่เชื่อจากใจจริงว่านักพัฒนาซอฟต์แวร์จะยังมีงานทำ
